## 项目背景
本项目是 AI Trade Customer Analyzer V3.2.4，
FastAPI + SQLAlchemy + SQLite 的本地单用户工具。

## 本次开发任务：客户地理分布地图页面

### 1. 数据库改动
在现有 Customer 模型新增三列：
- latitude: Float, nullable
- longitude: Float, nullable  
- geocode_status: String, default="pending"（枚举值：pending / done / failed）

### 2. Geocoding 服务（后端）
- 使用 geopy + Nominatim 对 Customer 表的 city + country 字段进行地理编码
- 限速 1次/秒（用 RateLimiter 包装）
- 结果写入上述三列，不重复处理 geocode_status="done" 的记录
- 提供一个后台 batch 任务接口：POST /api/customers/geocode/batch
- 提供单条触发接口：POST /api/customers/{id}/geocode

### 3. 地图数据接口（后端）
- GET /api/customers/map
- 只返回有坐标的客户，字段：id, name, country, city, 
  latitude, longitude, total_score, status, priority
- 支持按 country 过滤：?country=Saudi Arabia

### 4. 前端地图页面
- 使用 Leaflet.js + OpenStreetMap 瓦片
- 使用 Leaflet.MarkerCluster 插件做点聚合
- Marker 点击后 Popup 显示：公司名、国家/城市、评分、状态
- 左侧或顶部加国家筛选下拉框
- 页面顶部显示统计：总客户数 / 已定位数 / 未定位数

### 5. 不需要做的
- 不需要实时 geocoding（批量跑一次即可）
- 不需要地图编辑功能
- 不需要导出功能
