使用说明
配置 API Key

在启动前设置环境变量 HUNTER_API_KEY： 

# Windows (cmd)
set HUNTER_API_KEY=your_key_here

# Windows (PowerShell)
$env:HUNTER_API_KEY="your_key_here"

# Linux/Mac
export HUNTER_API_KEY=your_key_here

测试时可使用 test-api-key（返回测试数据）。
获取正式 Key：https://hunter.io/api-keys

查找策略（额度优化）
  1. Email Count（免费） — 先查该公司有多少邮箱，如果 total=0 直接返回，不消耗搜索额度。
  2. 本地缓存优先 — 所有结果缓存 7 天，相同域名+参数的查询直接从缓存读取。
  3. Domain Search 自带验证 — 域名搜索结果含 verification，不额外调用 Verifier。
  4. 智能降级 — 已知姓名时先在 Domain Search 结果中匹配，找不到才触发 Email Finder。
  5. 结果过滤 — 可在客户端按部门/级别筛选，无需多次 API 调用。
集成到客户详情页

进入任意客户详情页 → 点击 Hunter 查邮箱 按钮 → 系统自动使用该公司域名查询 Hunter 数据库 → 可在输入框中填写联系人姓名进行精确查找。

也可在导航栏点击 Hunter 邮箱 进入独立查找页面，直接输入域名和姓名进行搜索。
