# AI客户开发系统 V2.5大更新 功能扩展方案

## 基于公司网址的相似客户扩展模块（V1简化版）

---

# 1. 功能目标

用户输入一个目标公司网址后，系统自动分析该公司业务内容，并在指定国家范围内搜索相似公司，生成潜在客户列表。

用于外贸开发中的“种子客户扩展”。

---

# 2. 输入设计

## 2.1 输入参数

| 参数             | 类型     | 说明     |
| -------------- | ------ | ------ |
| company_url    | string | 目标公司官网 |
| target_country | string | 目标国家   |

## 2.2 示例

```json
{
  "company_url": "https://example.com",
  "target_country": "Mexico"
}
```

---

# 3. 输出设计

## 3.1 输出结构

```json
{
  "seed_company": {
    "name": "",
    "industry": "",
    "keywords": []
  },
  "similar_companies": [
    {
      "name": "",
      "website": "",
      "country": "",
      "similarity_score": 0.0
    }
  ]
}
```

---

# 4. 功能流程设计

## 4.1 总流程

URL输入
↓
网页抓取
↓
LLM提取业务信息
↓
关键词 & 行业识别
↓
国家范围搜索
↓
简单相似度排序
↓
输出结果

---

## 4.2 详细步骤

---

## Step 1：网页内容抓取

### 输入

* company_url

### 处理

抓取以下内容：

* title
* meta description
* h1-h3标签
* 产品/服务正文

### 输出

原始文本内容

---

## Step 2：业务信息提取（LLM）

从网页内容中提取：

* 主营行业
* 产品/服务
* 应用场景
* 核心关键词

### 输出示例

```json
{
  "industry": "Water Treatment Equipment",
  "products": [
    "industrial filtration system",
    "RO water purifier"
  ],
  "keywords": [
    "water treatment",
    "filtration",
    "industrial water system"
  ]
}
```

---

## Step 3：行业归类

将关键词映射为标准行业标签：

| 关键词               | 行业    |
| ----------------- | ----- |
| water treatment   | 水处理设备 |
| packaging machine | 包装机械  |
| solar panel       | 光伏设备  |

---

## Step 4：同国家公司搜索

在目标国家范围内进行关键词搜索：

### 搜索规则

* industry + country
* product + country
* keyword + companies + country

### 示例

* water treatment Mexico companies
* industrial filtration Mexico suppliers

---

## Step 5：相似度计算（简化规则）

### 评分模型

| 维度     | 权重  |
| ------ | --- |
| 关键词匹配  | 60% |
| 行业一致性  | 30% |
| 网站内容相似 | 10% |

### 公式

```text
score = 0.6 * keyword_match
      + 0.3 * industry_match
      + 0.1 * content_similarity
```

---

## Step 6：结果输出

返回 Top N 公司（建议 20–50）

每条数据包含：

* 公司名称
* 官网
* 国家
* 相似度评分

---

# 5. 系统架构（V1简化版）

```text
[输入 company_url]
        ↓
[网页抓取模块]
        ↓
[LLM业务信息提取]
        ↓
[关键词 + 行业分类]
        ↓
[搜索引擎查询（国家限定）]
        ↓
[规则相似度计算]
        ↓
[结果排序]
        ↓
[输出客户列表]
```

---

# 6. 技术实现方案

## 6.1 推荐技术栈

| 模块  | 技术                    |
| --- | --------------------- |
| 爬虫  | Playwright / Requests |
| LLM | DeepSeek / GPT        |
| 搜索  | Bing API / SerpAPI    |
| 后端  | Python + FastAPI      |

---

## 6.2 数据处理方式

V1版本采用：

* 关键词匹配（非向量模型）
* 简单规则评分
* 搜索引擎结果过滤

不使用：

* embedding向量数据库
* 复杂语义模型
* 公司知识图谱

---

# 7. V1版本限制

本版本限制如下：

* 不支持语义相似度模型（embedding）
* 不构建公司数据库
* 依赖搜索引擎结果质量
* 相似度计算为规则模型
* 数据覆盖范围有限

---

# 8. 适用场景

* 外贸客户开发
* 同行业客户扩展
* 竞争对手分析
* 新市场快速冷启动

---

# 9. 后续升级方向（V2）

后续可升级为：

* embedding语义相似度匹配
* 公司向量数据库（pgvector）
* 自动构建行业客户图谱
* 多国家批量扩展
* CRM自动导入系统
* 自动邮件开发模块

---

# 10. 总结

V1版本目标是：

实现“可运行的客户扩展最小闭环”，
以低成本方式验证“种子客户 → 相似客户扩展”逻辑。
